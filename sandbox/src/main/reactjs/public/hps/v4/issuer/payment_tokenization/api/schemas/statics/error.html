Error in Connect'API

This page provides a comprehensive reference of Core API error codes. You can use this information to diagnose failed transactions and fine-tune your exception-handling capabilities.

HTTP response codes

The Connect API uses standard HTTP response codes to indicate the status of requests:

  • Codes in the 2xx range typically indicate success.
  • Codes in the 4xx range typically indicate an error in the request.
  • Codes in the 5xx range typically indicate an error at the PowerCard platform.

The following table shows some common technical response codes:

Error code Description
00000 Success
99999 System error
90009 Generic error for request validation (check size,format,mandatory etc..)
00002 Forbidden you don't have permission to access for this api
00007 Authorization header is empty
00008 Authorization header is missing
00009 Malformat Authorization content
00010 Bearer is required in the token
00011 Token expired
00012 Invalid token
