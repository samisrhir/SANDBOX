Currency Management

The currency codes are sent and received in transaction and interchange files, etc. indicating the currency of related financial records. It is possible to declare several currencies according to the requirements of the concerned Financial Institution.

  • Alpha code currency Currency ISO alpha code in 3 letters based on the card industry standard. This field is mandatory. For example:
    • USD for United States Dollars;
    • EUR for Euro.
  • Numeric code Currency ISO numeric value on 3 digits, based on the card industry standards. This field is mandatory. For example:
    • 840 for United States Dollars;
    • 978 for Euro.
  • Exponent Number of decimal places in any amount expressed in a given currency. It is used by the system for printing and displaying format purposes.

Currency in requests

The nominal amount value is divided by 10n where n is the currency exponent of the currency code sent in the field below the last n digits are used for decimals.
Example An amount received as 9999999 will be handled by the connectAPI as US$ 99,999.99.

Currency in responses

The nominal amount value is multiplied by 10n where n is the currency exponent of the currency code sent in the field below the last n digits are used for decimals.
Example An amount that should be displayed as US$ 99,999.99 will be sent by the connectAPI as 9999999.
